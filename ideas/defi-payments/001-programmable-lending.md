# 可编程贷款协议 / Programmable Lending Protocol

> 贷款条件写入 Move 合约，还款、清算、利率调整全自动，无需信任第三方

## 解决什么问题

传统借贷依赖中心化机构审批和管理，效率低且存在信任风险。借款人需承担不透明的利率和不合理的清算条件。通过将贷款逻辑完全编码到 Move 合约中，可以实现无需信任第三方的自动化借贷，降低违约风险并提升资金效率。

## 核心功能

- 自定义贷款条件：抵押率、利率模型、还款周期全部参数化写入合约
- 自动化清算引擎：当抵押率低于阈值时触发渐进式清算，保护借贷双方利益
- 动态利率调整：根据资金利用率实时调整借贷利率，基于 KTYC（Kinked Token Yield Curve）模型
- 多资产抵押支持：支持 SUI、USDC、BTC 等多种资产作为抵押品

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | 贷款仓位作为独立 Object，支持精细化所有权管理 |
| PTB (Programmable Transaction Block) | 原子化执行借贷、还款、清算等复合操作 |
| Coin 标准 | 统一处理多种抵押资产和借贷资产 |
| 动态字段 (Dynamic Fields) | 存储贷款元数据和利率历史记录 |
| 共享对象 (Shared Object) | 借贷池作为共享对象，允许多用户并发交互 |

## 实现方案

### 智能合约 (Move)
- `lending_pool` 模块：管理资金池的存取、利率计算
- `loan_manager` 模块：创建、管理贷款仓位（NFT 化）
- `price_oracle` 模块：集成 Pyth/Suins 预言机获取资产价格
- `liquidation` 模块：清算逻辑，支持部分清算和闪电清算
- 关键数据结构：`LoanPosition` 包含抵押品、借入资产、健康因子、创建时间等

### 前端 / 后端
- 技术栈：Next.js + @mysten/dapp-kit + TailwindCSS
- 关键页面：Dashboard（总览）、Borrow（借款）、Lend（放款）、Liquidation（清算监控）
- 后端用 Sui TypeScript SDK 监听链上事件，计算实时利率

## 技术架构

用户通过前端发起借款请求，PTB 原子化执行：锁定抵押品 → 检查健康因子 → 从资金池借出资产。预言机每 epoch 更新价格，清算机器人监控所有仓位。LoanPosition 以 NFT 形式归用户所有，用户可通过 PTB 增减抵押或部分还款。所有状态变更通过事件通知前端。
