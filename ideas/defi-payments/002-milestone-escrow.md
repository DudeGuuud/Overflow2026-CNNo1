# 里程碑式托管 / Milestone-Based Escrow

> 资金锁在对象中，按里程碑条件（时间 + 外部验证）自动释放，适用于外包、贸易

## 解决什么问题

传统外包和贸易交易中，买卖双方缺乏互信：买方担心付了款收不到货，卖方担心交了货收不到钱。现有托管方案依赖中心化平台，费用高且不透明。里程碑式托管将资金锁定在 Sui 对象中，根据预设的条件（时间锁定、外部验证者确认）自动释放资金，降低信任成本。

## 核心功能

- 创建多里程碑托管合约：定义 N 个阶段，每阶段绑定金额和条件
- 多方验证机制：可指定独立验证者确认里程碑完成
- 时间锁定释放：超时自动释放给指定方，防止恶意拖延
- 争议解决通道：触发仲裁流程暂停资金释放

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | EscrowTicket 作为独立 Object，所有权在合约控制中 |
| PTB | 原子化执行"验证确认 + 资金转移"复合操作 |
| 时间锁 (Epoch) | 基于 Sui epoch 实现里程碑超时机制 |
| 动态字段 | 存储里程碑详情、验证者列表和状态 |
| TransferPolicy | 定义资金释放规则和权限 |

## 实现方案

### 智能合约 (Move)
- `escrow` 模块：创建托管合约，锁定资金到 EscrowTicket 对象
- `milestone` 模块：定义里程碑结构体，包含条件、金额、验证者、状态
- `verifier` 模块：管理验证者注册和签名验证
- `dispute` 模块：争议提交和仲裁流程
- 关键数据结构：`EscrowTicket { milestones: vector<Milestone>, parties: (address, address), created_at: u64 }`

### 前端 / 后端
- 技术栈：React + @mysten/dapp-kit + Sui TypeScript SDK
- 关键页面：Create Escrow（创建托管）、Dashboard（查看所有托管）、Milestone Detail（里程碑详情和验证）
- 后端服务监听链上事件，发送里程碑到期提醒通知

## 技术架构

买方创建托管合约时通过 PTB 将资金锁定到共享对象中。每个里程碑包含条件哈希和验证者地址。当里程碑条件满足时，验证者通过 PTB 提交确认，触发部分资金释放给卖方。如果 epoch 超时未确认，买方可申请退款。所有操作记录在链上事件中，前端实时订阅展示。
