# Liquidation Bot / 清算机器人

> 监控 DeepBook Margin 可清算仓位，自动执行清算赚取奖励

## 解决什么问题

DeepBook Margin 协议中的不健康仓位需要及时清算以保护协议安全，但手动监控效率低。清算机器人自动扫描所有 Margin 仓位，发现可清算仓位时第一时间提交清算交易，赚取清算奖励，同时保护协议免受坏账风险。

## 核心功能

- 实时监控所有 DeepBook Margin 仓位的健康度
- 当仓位健康度低于清算阈值时自动提交清算交易
- Gas 竞价：确保清算交易优先被打包
- 清算收益统计和策略优化

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| deepbook_margin | 监控仓位健康度、执行清算 |
| predict::mint | 清算后资产可投入 Predict 策略 |
| predict::redeem_permissionless | 清算收益的 Predict 投资赎回 |
| deepbook 现货 | 清算后资产兑换 |

## 实现方案

### 智能合约 (Move)
- `liquidation_rewards` 模块：记录清算贡献和奖励分配
- 关键数据结构：`LiquidationRecord`（仓位 ID、清算时间、奖励金额）

### Bot / Keeper / 服务端
- 仓位扫描器：通过索引器实时获取所有 Margin 仓位状态
- 健康度计算器：根据预言机价格实时计算每个仓位的健康因子
- 清算执行器：发现可清算仓位时立即提交清算 PTB
- Gas 优化器：动态调整 Gas 价格确保优先打包
- 收益管理器：清算奖励自动复投或提取

### 前端 / 后端
- 技术栈：React + WebSocket
- 实时仓位监控面板（健康度热力图）
- 清算历史和收益统计
- 竞争状态展示（其他清算者活动）

## 技术架构

Bot 通过索引器 WebSocket 监听 Margin 仓位变化 → 健康度计算器实时评估 → 发现健康度低于阈值的仓位 → 清算执行器构建 PTB（包含清算调用）→ 提交到 Sui 网络 → 清算奖励分配。整个过程毫秒级响应。
