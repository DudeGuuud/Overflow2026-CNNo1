# Market Making Bot / 做市机器人

> 在 Predict 上持续双边报价，赚取买卖价差，自动管理库存风险

## 解决什么问题

Predict 市场需要流动性提供者来缩小买卖价差，但手动做市风险高且效率低。做市机器人根据 OracleSVI 曲线自动计算理论价格，在 Predict 上持续双边报价（买入和卖出区间），赚取买卖价差的同时自动管理库存风险。

## 核心功能

- 基于 SVI 曲线自动计算各区间理论价格
- 持续在热门行权价和到期日双边报价
- 库存管理：仓位偏斜时自动调整报价（偏多时降低买价）
- 风险控制：最大库存限制、单笔最大暴露、亏损熔断

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| predict::mint | 双边报价（买入和卖出区间） |
| predict::redeem_permissionless | 对冲仓位到期后赎回 |
| predict::supply | 部分资金供应 PLP 赚取基础收益 |
| PredictManager | 管理所有做市仓位 |
| OracleSVI | 定价核心，计算理论区间价格 |

## 实现方案

### 智能合约 (Move)
- `mm_vault` 模块：做市资金管理、库存跟踪
- `quote_manager` 模块：记录活跃报价和成交
- 关键数据结构：`MMState`（库存、活跃报价列表、累计盈亏）、`Quote`（行权价、方向、数量、价格）

### Bot / Keeper / 服务端
- 定价引擎：基于 SVI 参数计算各区间的理论价格
- 报价管理器：根据理论价格 ± spread 生成双边报价
- 库存监控：实时计算净暴露，调整报价偏移
- 再平衡器：库存过大时通过 DeepBook 现货对冲
- 风控模块：连续亏损时自动暂停做市

### 前端 / 后端
- 技术栈：React + ECharts
- 做市面板：当前报价、库存状态、PnL
- 风险指标：最大暴露、库存偏斜度、实时盈亏

## 技术架构

定价引擎读取 OracleSVI → 计算理论价格 → 报价管理器生成双边报价 → 通过 predict::mint 提交 → 成交后更新库存 → 库存偏移时调整报价参数 → 到期时 redeem 结算。Bot 持续运行保持市场流动性。
