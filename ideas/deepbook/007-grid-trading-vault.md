# Grid Trading Vault / 网格交易金库

> 在 Predict 区间中部署价格网格，每个区间独立管理，到期自动重新部署

## 解决什么问题

链上缺乏系统化的网格交易工具。传统网格在 CEX 上运行，用户需要 KYC 且资金托管在中心化平台。Grid Trading Vault 将网格策略适配到 Predict 的区间结构上，每个价格区间对应一个独立的 Predict 仓位，到期自动重新部署，实现完全链上自动化。

## 核心功能

- 在用户设定的价格范围内均匀分布网格区间
- 每个网格节点对应一个 predict::mint 操作
- 到期自动检查每个区间的盈亏状态，盈利区间续期，亏损区间调整
- 可视化网格状态和每个节点的实时收益

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| predict::mint | 在每个网格节点 mint 区间期权 |
| predict::redeem_permissionless | 到期后赎回各区间赔付 |
| PredictManager | 管理全部网格节点的仓位 |
| OracleSVI | 优化网格间距（高波动区域更密） |

## 实现方案

### 智能合约 (Move)
- `grid_vault` 模块：网格配置、用户存款管理
- `grid_node` 模块：单个网格节点的状态管理
- 关键数据结构：`GridConfig`（价格上下限、网格数量、间距类型）、`GridNode`（行权价、数量、状态）

### Bot / Keeper / 服务端
- 网格部署器：根据配置计算所有网格节点并批量 mint
- 到期处理器：扫描所有节点，redeem 已结算的，重新部署新的
- 再平衡器：根据价格移动动态调整网格范围

### 前端 / 后端
- 技术栈：React + Canvas（网格可视化）
- 交互式网格编辑器：拖拽设定价格范围
- 实时网格热力图：每个节点着色表示盈亏状态

## 技术架构

用户配置网格参数 → 合约计算网格节点列表 → Keeper 批量提交 mint PTB → 每个节点独立到期结算 → Keeper 扫描并提交 redeem + re-deploy PTB → 前端通过索引器实时展示所有网格节点状态和总收益。
