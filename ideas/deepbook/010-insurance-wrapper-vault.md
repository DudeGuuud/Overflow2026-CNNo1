# Insurance Wrapper Vault / 保险封装金库

> 包装 Predict 二元期权为"价格保险"产品，用户买"BTC 不会跌破 X"的保单

## 解决什么问题

传统预测市场术语（二元期权、行权价、到期日）对普通用户不友好。保险封装金库将 Predict 期权重新包装为直观的"价格保险"产品，用户只需选择"保 BTC 不跌破 6 万"这种自然语言描述，后台自动转换为 Predict 合约操作。

## 核心功能

- 自然语言保险产品：用户选择保单类型（不跌破/不突破某价格）、保额、保期
- 后台自动将保单转换为 predict::mint 操作
- 保单到期自动理赔：如果触发了保险条件，自动赔付
- 保单市场：用户可以转让未到期的保单

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| predict::mint | 购买保险对应的二元期权 |
| predict::redeem_permissionless | 保险到期后自动理赔 |
| PredictManager | 管理所有保险保单 |
| OracleSVI | 自动计算保费（基于波动率） |

## 实现方案

### 智能合约 (Move)
- `insurance_vault` 模块：保单创建、保单代币化
- `policy_token` 模块：每份保单铸造为 NFT，可转让
- 关键数据结构：`Policy`（保险类型、触发价格、保额、到期日、状态）、`PolicyNFT`（可转让的保单凭证）

### Bot / Keeper / 服务端
- 保费计算器：根据 SVI 和到期日自动计算保费
- 理赔处理器：监控预言机价格，触发条件时自动 redeem
- 保单到期提醒：通知用户保单即将到期

### 前端 / 后端
- 技术栈：React + @mysten/dapp-kit
- 保险商店页面：预设热门保单（如"BTC 不跌破 6 万"）
- 自定义保单创建器
- 我的保单管理页面

## 技术架构

用户在前端选择保单 → 合约计算保费 → predict::mint 购买对应期权 → 铸造 PolicyNFT 给用户 → Keeper 监控预言机 → 触发条件时 redeem + 自动理赔 → 用户收到赔付。保单 NFT 可在二级市场转让。
